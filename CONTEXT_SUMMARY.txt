PROJECT OVERVIEW
----------------
A Next.js FIFA World Cup 2026 prediction app using:
- Next.js (App Router)
- NextAuth (Google + Email providers, Prisma adapter, database sessions)
- Prisma + SQLite
- Tailwind CSS
- Push notifications (web push)

User roles: USER | SUB_ADMIN | ADMIN
Group-based predictions: users join groups, predictions are scoped to a group


ARCHITECTURE SUMMARY
--------------------
- /src/app/             → Next.js pages (App Router)
- /src/app/api/         → API routes
- /src/components/      → Shared components (Navbar.tsx, etc.)
- /src/lib/auth.ts      → NextAuth config
- /src/lib/prisma.ts    → Prisma client singleton
- /prisma/schema.prisma → DB schema
- /prisma/seed.ts       → Official WC2026 fixture data (104 matches)


KEY SCHEMA MODELS
-----------------
User         → id, name, email, role, isDemo
Group        → id, name, description, avatar, createdBy, exactMatchPoints, directionMatchPoints
GroupMembership → userId, groupId, status (PENDING|APPROVED|REJECTED)
Prediction   → userId, matchId, groupId, homeScore, awayScore, points
Match        → matchNumber(1-104), homeTeam, awayTeam, group, round, venue, city, kickoff, homeScore, awayScore, status
CustomPrediction → groupId, question, options(JSON), points, lockTime, correctOption, status(OPEN|RESOLVED)
CustomPredictionAnswer → userId, customPredictionId, option, points
PointSettings → exactMatchPoints(5), directionMatchPoints(1)  [global default, now per-group too]
PushSubscription, MatchReminder, DemoSettings

KNOWN ISSUES / GOTCHAS
-----------------------
- After any `prisma db push`, run `npx prisma generate` to update local client
- After DB reset, admin role is set on SECOND sign-in (not first), because
  updateMany() fires before user record exists on first sign-in
- Browser cache can serve stale redirect pages — use hard refresh or incognito
  if redirects seem wrong after config changes
- SQLite: adding non-nullable columns via `db push` can recreate tables and
  wipe data (use --force-reset to intentionally reset)


HOW TO RESEED THE DATABASE LOCALLY
------------------------------------
git pull origin claude/fifa-prediction-app-WsAHW
npx prisma generate
npx prisma db push --force-reset
npx ts-node --compiler-options '{"module":"CommonJS"}' prisma/seed.ts
npm run dev

Then sign in with ADMIN_EMAIL (set in .env) to get admin role on second sign-in.
Go to /admin → Groups tab → create your first group.
Users can then browse /groups and request to join.
